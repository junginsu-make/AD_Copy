### 2. 파일 구조 확인

다음 폴더들이 보여야 합니다:

```
✅ app/                   - Next.js 앱 디렉토리
✅ components/            - React 컴포넌트

✅ src/                   - 비즈니스 로직
✅ public/                - 정적 파일
✅ scripts/               - 유틸리티 스크립트
✅ package.json           - 의존성 관리
✅ README.md              - 프로젝트 설명
✅ env.example.txt        - 환경 변수 템플릿
```

### 3. 보안 파일 확인 (중요!)

다음 파일들이 **없어야** 합니다:

```
❌ env.local.txt          - API 키 포함!
❌ node_modules/          - npm 패키지
❌ .next/                 - 빌드 파일
❌ *.log                  - 로그 파일
```

### 4. API 키 노출 확인

GitHub 저장소 검색창에서 다음을 검색:

```
OPENAI_API_KEY
```

**결과:**
- ✅ `env.example.txt`에서만 나와야 함
- ✅ 실제 API 키 값이 아닌 `your_openai_api_key_here` 같은 템플릿만 보여야 함

**만약 실제 API 키가 보인다면:**
1. 즉시 해당 파일을 저장소에서 제거
2. 노출된 모든 API 키를 즉시 재발급

### EC2 배포 요약

```bash
# 1. EC2 SSH 접속
ssh -i "your-key.pem" ubuntu@your-ec2-ip

# 2. Node.js 설치
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt update && sudo apt install -y nodejs git nginx
sudo npm install -g pm2

# 3. 저장소 클론
git clone https://github.com/junginsu-make/AD_Copy.git
cd AD_Copy

# 4. 환경 변수 설정
nano .env.local
# 실제 API 키 입력:
# NEXT_PUBLIC_SUPABASE_URL=...
# NEXT_PUBLIC_SUPABASE_ANON_KEY=...
# DATABASE_URL=...
# OPENAI_API_KEY=...
# ANTHROPIC_API_KEY=...
# GEMINI_API_KEY_1=...
# 등등

# 5. 의존성 설치
npm install

# 6. 빌드
npm run build

# 7. PM2로 실행
pm2 start npm --name "ad-copy-generator" -- start
pm2 startup
pm2 save

# 8. Nginx 설정 (선택사항)
# AWS_EC2_배포_가이드.md 참고
```
